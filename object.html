Object-Oriented Design

In Object-Oriented Design, a object is a collection of data with associated behaviors.
Classes describes objects. A class can define specific sets of characteristics that are shared by all objects from that class.
Classes have relation to each other. UML class diagrams describe classes and relationships.

Practical Activity: Creating an object-oriented design

There are some types of relationships in Object-Oriented Design.

  • Association : Relationship among two classes in any way.
  • Composition : The act of collecting several objects together to create a new one.
  • Aggregation : Almost exactly like composition. The difference is that aggregate objects can exist independently.
  • Generalization/Inheritance : Relation between a parent class and a child class that inherits attributes and methods from the parent.

    Database components

    A database is defined as a shared collection of logically related data and its description that is designed to meet the information needs of an organization. (David et al, 2016)
    Core elements of a database is the following.
  • Entities : It represents things which is objects in Objects-Oriented Design. It is tables in a databases.
  • Attributes : It is valuables or states within entities, which is columns in tables.
  • Relationship : Links among entities. It is relation connected with keys.
    • - Entities and relationships can be described by using Entity Relationship Diagram(ERD)
  • Primary Key : One or more columns that are used to provide a unique identifier for an individual record in a table.
  • Foreign Key : A column in a table that references a column in another table.

    • Normalization

    Normalization is a design process to reduce redundancy in the data structure and make tables desirable forms.

    The operation of Database

    The Database Management System(DBMS) is the essential technology to operate databases.
    DBMS is a software that interacts with the users' application programs and the database.
    It enables users to do the followings:

  • Create databases and tables with DDL(Data Definition Language)
  • Manipulate data in the database with DML(Data Manipulation Language)
    - The most common language SQL(Structured Query Language)
  • Provide managed access to the database in terms of security, consistency, shared access and recovery from database troubles.

    MVC Model

    MVC model is the well-known design architecture for web development. It consists of Model,View and Controller.

  • Model : Perform data access and business logic.
  • View : Perform user interface, which displays the data contained within Model. It includes UI components such as forms or buttons that users can see.
  • Controller : A connector between Model and View. It receives data from View and manipulates Model.

    Alternatives to SQL

    In a collaborative discussion of Unit 8, I researched the information about NoSQL(Not only SQL) as a alternative to SQL.

    Collaborative Discussion 2: Alternatives to SQL

    There is no necessity to define schemes and relations in the design of a NoSQL databases. It allows users to store more native data into a database, such as JSON, XML and any other types of objects. It is more suitable for manipulating a big data (Forbes, 2018).

    The major types of NoSQL databases is the following:

    TypeDescriptionDB
    Key ValueStore key/value form of data such as dictionary, hash or associative array.Redis, Riak, DynamoDB
    DocumentStore document based, unstructured data such as JSON, XML.MongoDB, DynamoDB, CouchDB
    GraphStore data in graphical structure. Suitable for a network type of data.Neo4j, CosmosDB, OrientDB
    ColumnInquiry data by column, not by row like RDB. Suitable for manipulating a big data.Cassandra, BigTable, HBase
    The common advantage of these technologies is the flexibility. There is a less necessity to design complex structure of logical tables and entity-relations like RDB to store unstructured data. Another merits of NoSQL are a high performance of manipulating big data and high extensibility(Jing et al, 2013). And also, it is developer-friendly since it enables users to insert data in a format used in applications into a database, such as JSON. Thus, NoSQL technologies allow users to build systems and databases quickly. It might be an effective way for startup businesses. It is also useful for building Data Lake in terms of its feature to store various types of unstructured data easily.

    On the other hand, most of NoSQL databases do not support a rich transaction function. It might be an obstacle to use NoSQL databases in a system that requires strict consistency and safety.

    In conclusion, NoSQL databases have advantages but it cannot replace traditional databases for the time being.

    This module covers the fundamental and application of Object-Oriented Design and Object-Oriented Programming. Before implementing Object-Oriented Programming, Object-Oriented Design must be applied properly. In Object-Oriented Design, a object contains data and operations to manipulate the data. Objects also represent things in a real world.(Ian, 2016)

    The Object-Oriented Design can be described by using UML(Unified Modeling Language) such as a class diagram. I learned the fundamental of UML by creating some class diagrams shown above sections, class diagrams describing a online super market or self-service checkouts at a supermarket. As written above, objects represent the real thing. It is said that Object-Oriented Design is more understandable because it can be related to things in a real world.(Ian, 2016) However, this does not mean we should create classes to all of things in a real world or scenarios to describe the system. Throughout this activity, it seems to me that the most difficult aspect of the design of Object-Oriented System is to define the scope of classes. (From my experience, I have not struggled with how to define the outline of the system this much in the design with procedural programming in terms of defining classes.) I defined some classes which is not necessary to implement the system but objects that appear only in the scenario.     Another point to be noted is readability. I reviewed outputs of some other students in practical activities, and I realized readability is as important as whether the design meets the requirements of the system. For example, small fonts should not be used in class diagrams. In addition, scaffolding codes such as getter and setter must no be included in the class as methods(2003, Scott ).

    Object-Oriented Programming

    This section examines Object-Oriented Programming. As shown in the above section, it has difficulties but some advantages as well.

    The followings are the core elements of Object-Oriented Programming, which are not seen in procedural programming such as C.

    Inheritance
    Polymorphism
    Encapsulation

    I created a web application to manage a local doctor's surgery with Python and Flask by applying those elements of Object-Oriented Programming, as an assignment in Unit 11.

    Throughout the activity, I realized the concept of Object-Oriented Programming. One of the advantages of Object-Oriented Programming is reusability of the code. Inheritance and Polymorphism enables developers to reduce lines of the codes to implement various functions such as below.

    class Employee(object):
      def __init__(self, name, address, phone):
        self.name = name
        self.address = address
        self.phone = phone

      def say_name(self):
        print(f'My name is {self.name}.')

    # This is an example of Inheritance.
    class Receptionist(Employee):
      def __init__(self, name, address, phone):
        super().__init__(name, address, phone)

    # This is an example of Inheritance.
    class WarehouseStaff(Employee):
      def __init__(self, name, address, phone):
        super().__init__(name, address, phone)

    def main():
    # This is an example of Polymorphism.
      receptionist = Receptionist('George', 'Essex, UK', '123456789')
      warehouse_staff = WarehouseStaff('Peter', 'London, UK', '987654321')
      receptionist.say_name()
      warehouse_staff.say_name()
    Without Inheritance, developers have to write same codes by a class and a function.

    The second merit is uniformity. Inheritance and Polymorphism can enhance uniformity of the code.

    from abc import ABCMeta, abstractmethod

    class Employee(metaclass=ABCMeta):
    # If get_name method is no implemented, raise TypeError.
      @abstractmethod
      def get_name(self) -> str:
        pass
    The example above shows every class inheriting Employee must implement get_name functions. This can prevent the code from the deviation from the implementation's rule of the project.

    The third positive point is flexibility. In Object-Oriented Programming, data and methods are encapsulated in a class. The data and methods are invisible from outside of the class. In addition, data can be manipulated only through accessor methods. Thanks to Encapsulation, the code based on Object-Oriented Programming can be flexible to change the implementation, which means a change in a class has less impacts on other classes. In fact, I changed my code several times in the activity, but I could save my time to confirm its impact.

    On the other hand, it seems to me that Object-Oriented Programming is more complex to implement than procedural languages. There are many things to learn before start implementing. It might be an obstacle for beginners to learn programming. Besides, it takes time to design the code. Developers have to consider the unit and the definition of classes first, such as what data and operations the class should have, or the valuables and the methods should be private or public and so on. Object-Oriented Programming has advantages as I wrote above. Even so, it is likely that it brings less merits to the development of simple applications.

    Overall, Object-Oriented Programming treats a set of code like a object or a component which has flexibility. It seems that this way of programming is more suitable for large-scale developments in which many developers are involved.
